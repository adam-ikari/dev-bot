# Dev-Bot TUI 用户指南

## 概述

Dev-Bot TUI（Terminal User Interface）提供现代化的终端用户界面，将终端分割为两个主要区域：

- **上方区域**: AI 运行日志显示区域（根据屏幕高度自动调整）
- **下方区域**: REPL 用户输入区域（根据屏幕高度自动调整）

### 自动布局调整

TUI 会根据终端屏幕高度自动调整两个区域的比例：
- **小屏幕**（< 20 行）: 日志 70%，REPL 30%
- **中等屏幕**（20-30 行）: 日志 75%，REPL 25%
- **大屏幕**（> 30 行）: 日志 80%，REPL 20%
- **最小保护**: REPL 至少保留 5 行
- **最大限制**: REPL 不超过 40%

屏幕大小变化时，布局会自动适应。

### 布局模式

TUI 支持两种布局模式：
- **自动模式**: 根据屏幕高度自动调整（默认）
- **全屏模式**: 可以切换到全屏日志或全屏 REPL 模式

## 启动 TUI 模式

```bash
dev-bot run --tui
```

## 功能特性

### 1. 日志显示

TUI 提供丰富的日志显示功能：

#### 日志类型

- **信息日志** (蓝色): 一般信息输出
- **成功消息** (绿色): 操作成功提示
- **警告消息** (黄色): 警告和提示
- **错误消息** (红色): 错误和异常
- **AI 输出** (青色): AI 工具的输出内容
- **调试信息** (灰色): 调试和详细日志

#### 日志滚动

- 使用 `↑` / `↓` 箭头键滚动日志
- 使用 `Page Up` / `Page Down` 快速翻页
- 使用 `Home` / `End` 跳转到开头/结尾

#### 日志缓冲

- 最多保留 1000 行日志
- 自动清理旧日志
- 支持日志搜索

### 2. REPL 输入

#### 输入方式

1. 在下方输入框中输入指令
2. 按 `Enter` 键发送
3. 输入会立即显示在日志区域

#### 支持的指令

- `help`: 显示帮助信息
- `status`: 查看当前状态
- `specs`: 列出所有 Spec 文件
- `run`: 运行开发任务
- `stop`: 停止当前任务
- `clear`: 清空日志

### 3. 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+C` | 退出 TUI |
| `Ctrl+Q` | 退出 TUI |
| `F1` | 切换全屏日志模式 |
| `F2` | 切换全屏 REPL 模式 |
| `↑` | 滚动日志（在日志区域） / 查看上一条历史输入（在 REPL 输入框） |
| `↓` | 滚动日志（在日志区域） / 查看下一条历史输入（在 REPL 输入框） |
| `Page Up` / `Page Down` | 快速翻页 |
| `Home` / `End` | 跳转开头/结尾 |
| `Enter` | 发送输入 |

## 历史输入

### 滚动历史输入

REPL 输入框支持历史输入滚动功能，方便用户查看和重复使用之前的输入。

#### 使用方法

1. **查看上一条历史输入**
   - 在 REPL 输入框中按 `↑` 键
   - 显示上一条输入
   - 可以继续按 `↑` 查看更早的历史

2. **查看下一条历史输入**
   - 在 REPL 输入框中按 `↓` 键
   - 显示下一条输入
   - 如果已到达最新历史，会恢复当前正在编辑的输入

3. **编辑历史输入**
   - 使用上下键选择历史输入后
   - 可以直接编辑历史内容
   - 按 Enter 发送修改后的内容

#### 特性

- **自动保存**: 所有提交的输入自动保存到历史记录
- **独立历史**: 历史记录独立于日志显示
- **编辑支持**: 可以编辑历史输入后重新发送
- **上下文保留**: 浏览历史时保留当前未提交的输入
- **光标位置**: 查看历史时自动将光标移到末尾

#### 注意事项

- 历史输入功能只在 REPL 输入框有焦点时生效
- 在日志区域按上下键会滚动日志，不影响历史输入
- 历史记录保存在内存中，退出 TUI 后会清空

## 自动布局调整

### 屏幕大小响应

TUI 会自动响应终端屏幕大小的变化：

- **终端启动时**: 根据初始屏幕高度计算合适的布局比例
- **调整终端大小时**: 自动重新计算并调整布局
- **实时适应**: 无需手动操作，布局始终保持最佳比例

### 调整规则

| 屏幕高度 | 日志区域 | REPL 区域 | 说明 |
|----------|----------|-----------|------|
| < 20 行 | 70% | 30% | 小屏幕，REPL 区域较大 |
| 20-30 行 | 75% | 25% | 中等屏幕，平衡布局 |
| > 30 行 | 80% | 20% | 大屏幕，日志区域更大 |
| 最小限制 | - | 5 行 | REPL 至少保留 5 行 |
| 最大限制 | - | 40% | REPL 不超过 40% |

### 实际效果

```

┌────────────────────┐
│                    │  10 行（67%）
│    日志区域        │
│                    │
├────────────────────┤
│    REPL 输入        │  5 行（33%）
└────────────────────┘

中等屏幕（25 行）:
┌────────────────────┐
│                    │  19 行（75%）
│    日志区域        │
│                    │
├────────────────────┤
│    REPL 输入        │  6 行（25%）
└────────────────────┘

大屏幕（40 行）:
┌────────────────────┐
│                    │  32 行（80%）
│    日志区域        │
│                    │
├────────────────────┤
│    REPL 输入        │  8 行（20%）
```

## 编程接口

### 基本使用

```python
from dev_bot.tui import DevBotTUI, TUILogger
import asyncio

async def main():
    # 创建 TUI 应用
    app = DevBotTUI()
    
    # 创建日志记录器
    logger = TUILogger(app)
    
    # 设置用户输入回调
    def handle_input(text: str):
        logger.info(f"收到输入: {text}")
    
    app.set_user_input_callback(handle_input)
    
    # 运行 TUI
    async with app.run_task():
        # 添加日志
        logger.info("启动应用...")
        logger.success("启动成功！")
        
        # 保持运行
        while True:
            await asyncio.sleep(1)

asyncio.run(main())
```

## 配置选项

### 环境变量

- `DEV_BOT_TUI_HEIGHT`: 日志区域高度（默认 70%）
- `DEV_BOT_TUI_MAX_LINES`: 最大日志行数（默认 1000）
- `DEV_BOT_TUI_THEME`: TUI 主题（默认使用 Textual 默认主题）

### 配置文件

在 `config.json` 中添加：

```json
{
  "tui": {
    "enabled": true,
    "log_height_percent": 70,
    "max_log_lines": 1000,
    "auto_scroll": true,
    "show_timestamp": true
  }
}
```

## 故障排除

### 问题 1: TUI 无法启动

**原因**: 终端不支持颜色或大小不足

**解决方案**:
```bash
# 使用支持 256 色的终端
export TERM=xterm-256color

# 确保终端大小足够（至少 24 行 x 80 列）
```

### 问题 2: 日志显示异常

**原因**: 日志编码问题或特殊字符

**解决方案**:
```python
# 使用 utf-8 编码
app.log("你的消息".encode('utf-8').decode('utf-8'))
```

### 问题 3: REPL 输入无响应

**原因**: 输入焦点不在 REPL 区域

**解决方案**:
- 点击下方输入框
- 或按 `Tab` 键切换焦点

## 最佳实践

1. **使用 TUI 模式**: 提供更好的用户体验
2. **定期查看日志**: 及时了解系统状态
3. **合理使用 REPL**: 通过 REPL 提供指令和反馈
4. **利用快捷键**: 提高操作效率
5. **保持终端大小**: 确保界面显示正常
